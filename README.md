# Mahangpt Frontend

این ریپازیتوری نسخه فرانت‌اند فروشگاه است و قرار است در ادامه به بک‌اند C# متصل شود. هدف ما تحویل یک فرانت تمیز، استاندارد، و آماده اتصال به API است.

## هدف پروژه
- آماده‌سازی صفحات برای اتصال به بک‌اند C# (API محور).
- استانداردسازی ساختار فایل‌ها، مسیرها و نام‌گذاری‌ها.
- نگه داشتن کد ساده، خوانا، و قابل توسعه برای تیم بک‌اند.

## ساختار پوشه‌ها
- `index.html` صفحه اصلی
- `pages/` سایر صفحات سایت (ورود، سبد خرید، پروفایل و ...)
- `assets/css/` استایل‌ها
- `assets/js/` اسکریپت‌های عمومی (در صورت نیاز داده محلی)
- `sandbox/` فایل‌های تست/دمو که در خروجی نهایی استفاده نمی‌شوند

## اصول کار (برای ایجاد صفحه جدید)
- صفحه جدید را داخل `pages/` بساز.
- لینک‌های داخلی به صفحه اصلی باید به صورت `../index.html` باشند.
- استایل و اسکریپت‌ها باید از مسیرهای زیر لود شوند:
  - `../assets/css/style.css`
  - `../assets/js/script.js`
- اگر فایل JS جدید لازم شد، در `assets/js/` قرار بده و فقط در صفحه مرتبط لود کن.
- نام فایل‌ها را تا حد امکان ساده و استاندارد نگه دار (ترجیحا lowercase و بدون فاصله).

## قراردادهای آماده‌سازی برای بک‌اند
- هیچ منطق بک‌اندی در فرانت پیاده نشود (فقط UI و رفتارهای ساده).
- فرم‌ها با ساختار واضح و قابل مپ شدن به API طراحی شوند (name/id مشخص).
- داده‌های نمونه (placeholder) باید قابل جایگزینی با داده API باشند.
- مسیرها طوری نوشته شوند که در آینده با روتینگ بک‌اند C# سازگار باشند.

## نکات اجرای محلی
- پروژه فعلا استاتیک است (HTML/CSS/JS) و بدون وابستگی اجرا می‌شود.

## یادآوری
هر بار که صفحه جدید ایجاد می‌شود، این فایل را بخوان و اصول بالا را رعایت کن تا خروجی نهایی یکدست و قابل تحویل باشد.


## بهینه‌سازی‌ها (نسخه بازطراحی‌شده)
این نسخه با حفظ دقیق UI/UX (پیکسل-به-پیکسل تا حد ممکن) نسبت به نسخه اولیه بهبودهای زیر را دارد:

- بهبود امنیت فرانت:
  - اضافه شدن Policyهای سبک در `<head>` (CSP/Permissions/Referrer) برای محدود کردن منابع و کاهش ریسک‌های رایج.
  - سخت‌سازی Toastها: حذف `innerHTML` برای رشته‌های داینامیک و استفاده از DOM امن.
  - افزودن `rel="noopener noreferrer"` به لینک‌های خارجی که `target="_blank"` دارند.

- بهبود کارایی و تجربه موبایل:
  - توقف خودکار اسلایدر بنر و اسکرولرهای خودکار محصولات وقتی خارج از دید هستند (IntersectionObserver) جهت کاهش مصرف CPU/باتری.
  - افزودن `width/height` و `decoding="async"` برای همه تصاویر جهت کاهش CLS و بهبود رندر.

- استانداردسازی و نگهداشت:
  - اصلاح و ایمن‌سازی بخش‌های حساس JS بدون تغییر در ظاهر.
  - سازگاری بهتر با حالت Reduced Motion.

نکته: پروژه همچنان استاتیک است و بدون وابستگی اجرا می‌شود.
